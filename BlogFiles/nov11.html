I watched this video to get a better understanding of useState.

  • Realized in his same code he had empty angle brackets as the parent element - didn't know you could do that!
  • React hooks can't be called conditionally, they have to be called in the same order every time
  • They can't be nested in functions or loops either
  • useState returns two values, the first is the state, and the second should be the function that updates the state:
  • const [count, setCount] = useState()
  • Functions that call setCount will update count
  • If you want to update count based on its previous value, pass it as a parameter before updating it
  • function decrement() {
                setCount(prevCount => prevCount - 1);
            }
  • Above will decrement count by 2. Not very intuitive.

  • useState can also take a function, which runs once.
  • When one part of a state is updated, the entire thing is overwritten. Spreading it avoids this issue.
  • Using multiple useState hooks makes it easier to manage
