I watched this video about react effects:

  • useEffect creates a side effect whenever something happens, such as when a specific resource changes
  • Can pass an array into useEffect that will indicate when it should run
    • useEffect(() => { ...
            }, [resource])
  • This is useful for fetching from an API whenever a resource changes, then the json can be added as a state, which can be coded into the HTML
  • Use a return statement to call a function that removes event listeners added by useeffect, so they aren't being added multiple times
  • This video explains that useEffect is asynchronous, but useLayoutEffect is synchronous, so it should be used when things are dependent on positions of the page, but may cause errors, so useEffect is preferred.

I watched this video about callback. useCallBack prevents a function from running unless a certain variable has changed, similar to useEffect. Not entirely sure why useEffect wouldn't just be used instead.
