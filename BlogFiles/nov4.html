It was unclear to me what the purpose of using React was over just plain Javascript, so I found an articlethat explained it:

  • Plain javascript means the HTML will be pre-set on the server, but with react, it is created on the browser
  • React keeps all the HTML information in one place, instead of splitting it between the HTML and JS files. This makes updating things easier.
  • Avoids manually extracting things like form input from DOM, saves in JS variables directly
  • Avoids having to set up listeners and append new elements to existing ones

React cuts down on the amount of code one has to write, as shown by the following example from the article:


        addButton.addEventListener("click", function() {
          const input = document.getElementById("item-input");
          const list = document.getElementById("grocery-list");
         
          const listNode = document.createElement("li");
          const textNode = document.createTextNode(input.value);
         
          listNode.appendChild(textNode);
          list.appendChild(listNode);
        });

function addItem() { setItems([...items, itemInput]); }
