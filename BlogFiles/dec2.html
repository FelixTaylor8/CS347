I was having some issues that I thought useLayoutEffect might solve, so I looked at an articlethat explained it:

  • useEffect and useLayoutEffect function almost identically, and their format is identical
  • This was an example from the article:
    • useLayoutEffect(() => {
                // do side effects
                return () => /* cleanup */
              }, [dependency, array]);
  • useEffect runs after components are rendered and the screen is updated
  • useLayoutEffect runs after components are rendered but before the screen is updated
  • useEffect is preferred most of the time, as most changes don't need to happen immediately
  • useLayoutEffect should be used when the page flickers - that's the issue I was getting
  • useLayoutEffect might cause performance issues

Changing to useLayoutEffect didn't actually solve the flickering problem I was getting, so I guess the author lied.

This article also said only to use useLayoutEffect to prevent flicker. So, I don't understand why it didn't fix it for me :[

It mentioned refs, so I looked at a video about it

  • Refs are similar to states because they persist between renders, but they won't reupdate the components when they're changed
  • useRef returns an object with a single property of "current"
    • 
                useEffect(() => {
                    renderCount.current = renderCount.current + renderCount.current + 1;
                })
  • This might be a way to deal with the flicker problem, though the flicker occurred when navigating between pages.
  • Refs can also be used to refer to HTML elements
