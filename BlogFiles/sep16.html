I did some practice with position elements using this webpage. Here are some notes:

  • position: relative; is used to contain position: absolute;
  • position: relative; is used to shift elements left and right
  • position: fixed; is always on the same position on the screen
  • top, bottom, left, and right affect where on the page the item is located
  • position: absolute; puts items in specific places
  • position: absolute; items do not stack
  • position: absolute; elements move relative to a non-static parent

It was cute, though I don't think it'll help me remember which is which. I always mix them up.

I was a little confused why it said that position: absolute; doesn't stack. I looked at other examples, such as on this site, and it looks like they do stack.

According to this site, stacking is when elements are piled up on top of each other. So what did the sundae site mean when it said they didn't stack, when it looks obvious that they do? I think they might have meant that position; absolute doesn't stack side-by-side, and they just used the wrong term. Odd.

I also followed this video tutorial to set up the navbar for my project. There was a lot of stuff shown in the video that I didn't want to use, but this aspect was cool:


        ul li a {
            transition: all 0.3s ease 0s;
        }

This slowed down the transition between the regular link color and the color it changes to when the mouse hovers over it. "all" indicates it should apply to all transition types (eg changing color), 0.3s is how long the transition should take place, and ease "specifies a transition effect with a slow start, then fast, then end slowly" (source). I wasn't too sure if the 0s was needed, but removing it slowed the transition down for some reason. Other examples didn't include it for ease.

Additionally, the video showed that


        ul li {
            display: inline-block;
        }

makes the list elements sit next to each other instead of above/below each other.
