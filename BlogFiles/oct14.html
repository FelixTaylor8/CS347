I watched this video to brush up on using "fetch" with an API. It's kinda difficult for me to learn code by reading textbooks and articles, so having a video tutorial helps a lot. I'm thinking of doing a Pokemon API for the project, so my example code revolves around that.

Following the video tutorial:

              fetch('https://pokeapi.co/api/v2/pokemon/sylveon')
              .then(res => res.json()),
              .then(data => console.log(data));

This spits out information about Sylveon.

If the Pokemon name was invalid, it would return {}

Use res.ok to ensure the resulting json is useable

            fetch('https://pokeapi.co/api/v2/pokemon/sylveon')
            .then(res => {
                if (res.ok) {
                    console.log('SUCCESS');
                } else {
                    console.log('Fail');
                }
            })
            .then(data => console.log(data))
            .catch(error => console.log('ERROR'));

He demonstrates POST, which I can't do with the API I'm using.

Also watched this video for further explanations on APIs. Here are some notes:

  • Application Programming Interface: what two applications use to communicate data to each other
  • JSON: JavaScript Object Notation - file uses JS objects with quotations around the element names
  • Use the documentation of an API if available
  • If https://pokeapi.co/api/v2/pokemon-species/sylveon is parsed, then looking through the flavor_text_entries list for the object where name="en" will provide a Pokedex entry
  • Javascript can allow pairs to be delcared together, e.g. const {name, order} = data;
