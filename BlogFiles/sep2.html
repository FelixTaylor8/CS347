Since I took CS280 last fall, a lot of this is review for me. To refresh my HTML memory, I watched a tutorial for setting up this site to look relatively nice. It helped me remember how CSS works, and that's useful. It refreshed my memory on using names and IDs so instruct CSS to present different elements in different ways, such as changing font size/color and adding a background image. For example, the header is called "#showcase" and the style sheet adds a background image and ensures it stays stable when the page is resized:

#showcase {
            background-image:url("https://wallpaperaccess.com/full/3581467.jpg");
            background-size:cover;
            background-position:center;
            height:100vh;
            display:flex;
            flex-direction:column;
            justify-content:center;
            align-items:center;
            text-align:center;
            padding: 0, 20px;
        }

If I wanted to interact with each "h1" element within showcase, I would use "#showcase h1" in the CSS declaration.

I figured the difference between "id" and "class" was that id could only apply to one element, while class could apply to multiple, and this page confirmed that.

Also, "vh" makes sure the image is the same height relative to that of the device it's viewed on. 100 pixels is always 100 pixels, but 100vh is different depending on how large a screen is.

What I'm not yet (and never really was) sure about is the display and flex attributes. Those always seemed confusing and buggy to me, they never looked right. I would just try random ones until I got something that looked okay.

Text alignment and padding are straightforward. The code puts it on the center of the page, and puts 20 pixels around it.

I wasn't sure what the difference between justify-content and align-items was, but former is for horizontal alignment and the latter is for vertical, according to this article.
